## Vision

### 10x Check
Not a "content aggregation system" — a **Digital Twin** that understands the relationships
between your experiences, thoughts, and connections. It doesn't just store what you said —
it understands what you're thinking. Cross-source, cross-time correlation engine that
surfaces insights you'd never find on your own.

### Platonic Ideal
"My AI knows my knowledge system better than I do. When I ask a question, it finds clues
from my past 6 months that I've forgotten. It feels like having a counselor with perfect
memory." Core shift: from "I go search" to "it tells me."

## Accepted Scope

### Core (from original description)
- Multi-source data import pipeline (WeChat manual paste/screenshot, Get Notes API, Photos)
- AI cleaning and distillation via Claude API
- Structured storage optimized for Agent consumption (Markdown + YAML frontmatter)
- Staging mechanism for hallucination prevention (high-confidence auto, low-confidence manual review)

### Expansions
- Knowledge Graph: entity extraction (people/companies/projects/locations) + relationship mapping
- Timeline & Pattern Recognition: attention shift tracking, emerging interest detection
- Relationship Network: people graph, interaction frequency, topic mapping
- Proactive Insight Engine: weekly summaries, follow-up reminders, opportunity alerts
- Photo Deep Understanding: multimodal scene description, whiteboard OCR, business card extraction
- Serendipity Engine: cross-source, cross-time unexpected correlation discovery
- Contextual Smart Reminders: context-enriched reminders with relevant background

## Architecture

### Approach: C — Agent-Native Knowledge Base
- **Storage:** Local file system (Markdown + JSON), no database initially
- **Language:** Python
- **AI Engine:** Claude API (text + multimodal)
- **Data Sources:** Get Notes (biji.com OpenAPI), WeChat (manual paste/screenshot), Photos (local)
- **Security:** Fully local, FileVault encryption, API keys in .env
- **Graph Format:** JSON-LD for entity relationships

### Phased Rollout
1. **Phase 1:** Get Notes import + basic distillation (validate pipeline)
2. **Phase 2:** WeChat text import + entity extraction
3. **Phase 3:** Photo import + multimodal processing
4. **Phase 4:** Knowledge graph + relationship network
5. **Phase 5:** Intelligence layer (patterns, serendipity, reminders)

## Key Design Decisions
- **Pure file system** (no SQLite) — revisit at ~5000 items
- **Staging mechanism** for AI hallucination prevention
- **Entity disambiguation** via context + staging review
- **Batch async distillation** (not synchronous with import)

## NOT in Scope
- Web UI (Agent-native design)
- Cloud deployment / multi-device sync
- Real-time WeChat monitoring
- Voice input (Phase 2)
- Calendar / email integration (Phase 2)
- Multi-user collaboration (Phase 3)

## Critical Gaps to Address in Implementation
1. **Entity disambiguation** — same name different people, same person different names
2. **Import friction** — WeChat manual import must be as low-friction as possible ("quick inbox" design)
3. **API cost management** — Claude API usage grows linearly with data volume
